# JUnit Test Workflow

## Agents and Steps

1. **Junity**
   - Ingests the Java Sling Model implementation.
   - Generates a single, compilable JUnit test class using AEM Mocks.
   - Covers `@ValueMapValue` getters, collection-building logic in `@PostConstruct`, and `getExportedType()`.

2. **junit_update**
   - Takes two Java files: a Sling Model implementation and an existing JUnit test class.
   - Updates or merges the test cases to ensure complete coverage.

3. **Git_Up**
   - Uploads code files generated by the agents to the GitHub repository.

## Workflow Execution

- Provide the Java Sling Model file to **Junity**.
- Junity generates the initial JUnit test class.
- Provide both the Sling Model file and existing test file to **junit_update**.
- junit_update merges/updates the test cases.
- Use **Git_Up** to upload the generated/updated test files to the repository under `Junit_Test`.

## Notes
- The `downloadjunit` agent has been removed from this workflow.
